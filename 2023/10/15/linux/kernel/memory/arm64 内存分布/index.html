ARM64 内存分布

ARM64 架构处理器采用 48 位的物理寻址机制，最多可以寻址 256TB 物理地址空间。虚拟地址也最高支持 48 位寻址。所以在处理器设计架构上，把虚拟地址空间划分为两个空间，用户空间和内核空间，并且每个空间最多寻址 256TB。

  • App space：0x00000000_00000000 ~ 0x0000FFFF_FFFFFFFF
  • Kernel space：0xFFFF0000_00000000 ~ 0xFFFFFFFF_FFFFFFFF

linux 内核中和内存布局相关的文件为 arch/arm64/include/asm/memory.h

PAGE_OFFSET 是物理内存在内核空间里做线性映射的起始地址，也就是物理的起始地址 + PAGE_OFFSET = 内核空间虚拟线性空间的起始地址。linux 内核会在初始化时把所有物理内存全部映射到内核空间的线性映射区

KIMAGE_VADDR 是内核映像文件映射到内核虚拟空间的起始地址。它等于内核模块区域的虚拟地址的结束地址 MODULES_END

编译器编译链接内核后就确定了映像的大小。其中的布局是由 arch/arm64/kernel/vmlinux.lds.S 来决定。包括：

段 起始地址 结束地址 描述
代码段（.text） _text _etext 包含了编译后的内核代码
init 段 __init_begin __init_end 包括了大部分模块初始化数据
数据段（.data） _sdata _edata 包含了内核中大部分变量
bss 段 __bss_start __bss_stop 包含了未初始化或初始化为 0 的全局变量和静态变量

上面这些段的大小在编译链接的时候根据内核配置就确定了，每种配置的代码段和数据段的长度都不相等，比如有多少内核模块。但是起始地址 _text 总是相同的。内核编译完之后会生成一个 System.map 文件，从这里就可以查到这些段的具体链接地址，即虚拟地址。或者也可以从 /proc/kallsyms 中获取。

已知内核映像文件映射到内核虚拟空间的起始地址为 KIMAGE_VADDR，而代码段的起始 _text 是由链接器脚本控制的。

代码段的起始地址等于 KIMAGE_VADDR + TEXT_OFFSET，而 TEXT_OFFSET 为代码段相对起始地址的偏移，其定义在 arch/arm64/Makefile 中，如果配置了 CONFIG_ARM64_RANDOMIZE_TEXT_OFFSET，它就是个随机数，每编译一次，启动时存放内核映像的位置就变一次，这是为了防止黑客攻击。没有配置的化就是默认值 0x00080000。

计算得到的值和在 System.map 中看的一样。

内核映像文件映射到了 vmalloc 区域中，这是为了实现 KASLR 特性。

KASLR 是 kernel address space layout randomization 的缩写，即内核地址空间布局随机化。这个技术允许内核映像加载到 vmalloc 区域的任何位置。

  • 当 KASLR 关闭的时候，每次开机内核映像都会映射到一个固定的链接地址，黑客很容易攻破。
  • 当 KASLR 打开的时候，每次开机内核映像都会映射到相对于链接地址不同偏移量的地址。

linux 内核提供了一个 dump 页表的调试接口。打开 CONFIG_ARM64_PTDUMP_DEBUGFS=y 配置后，然后 cat /sys/kernel/debug/kernel_page_tables 就可以打印出上面这些虚拟地址空间，并且还显示出每个区间的页表大小，页表的属性，内存属性等信息。

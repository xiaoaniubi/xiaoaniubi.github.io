内存类型和属性

ARMv8 架构提供两种内存类型：

  • 普通内存（NORMAL）
  • 设备内存（DEVICE）

普通内存

普通内存（如 RAM、Flash、ROM）是弱一致性的，限制很少，提供了最高的处理器性能。通常代码段、数据段等大部分数据都可以放到普通内存中。处理器可以对普通内存进行重排、重复和合并操作。普通内存可以让处理器做很多硬件优化，如分支预测、数据预取、高速缓存行预取和填充、乱序加载等。

共享属性

一个 SOC 中，除了 CPU 之外，还有其他可以访问内存的 Master 硬件单元，如 DMA、GPU 等，这些硬件单元可以称为 CPU 之外的观察点。在一个多核系统中，CPU、DMA、GPU 都通过系统总线连接到内存，它们都可以通过系统总线访问内存。

  • 如果一个内存区域被标记为不可共享的（Non-shareable），那只能被一个处理器访问。
  • 如果一个内存区域被标记为内部共享的（Inner Shareable），那它可以被多个处理器访问，但系统中其他的可访问内存的硬件单元就不能访问了，如 DMA、GPU 等。
  • 如果一个内存区域被标记为外部共享的（Outer Shareable），那系统中很多访问内存的硬件单元都可以访问，如 CPU、GPU、DMA 等。

每个 Inner(Outer) Shareable 域都包含一组观察器，对于该集合的任何成员使用 Inner(Outer) Shareable 属性进行的数据访问，这些观察器对于该集合的每个成员都是数据一致的。也就是说在内（外）部共享域中，一个成员对某个地址数据的改变都可以被其他成员看到，保持此共享域中缓存一致性。

内部共享域中的所有观察者都是同一外部共享域中的成员。这意味着内部可共享性域是外部可共享性域的子集，尽管它不一定是一个适当的子集。

因为对 Non-cacheable 位置的所有数据访问对于所有观察者来说都是数据一致的，所以 Non-cacheable 位置总是被视为 Outer Shareable。

缓存属性

除了上面的共享属性外，普通内存的每个区域都被分配了下面的其中一种 Cacheability 属性：

  • 直写，可缓存（Write-Through Cacheable）
  • 写回，可缓存（Write-Back Cacheable）
  • 不可缓存（Non-cacheable）

设备内存

处理器访问设备内存有很多限制，如不能进行预测访问等。设备内存是严格按照指令顺序来执行的，一般是留给外设来访问的，如 UART、SPI 等。

属性

ARMv8 架构定义了多种设备内存属性：

  • Device-nGnRnE：有更多的限制
  • Device-nGnRE
  • Device-nGRE
  • Device-GRE：有最少的限制

Device 后面的字母有特殊意义：

  • G 和 nG：分别表示聚合（Gathering）和不聚合（non Gathering）。聚合表示在同一个内存属性区域中允许把多次内存访问的操作合并为一次总线传输。
    • 若一个内存地址标记为 nG，会严格按照内存访问的次数和大小来操作，不会合并
    • 若一个内存地址标记为 G，则会做合并优化，如合并两个相邻字节访问为一次多字节访问。若程序访问同一个内存地址两次，那么处理器只会访问内存一次，但是和读两次内存的结果是一样的。若内存区域标记为 nG，那么处理器会读两次内存。
  • R 和 nR：分别表示指令重排（Re-ordering）和不重排（non Re-ordering）
  • E 和 nE：分别表示提前写应答（Early Write Acknowledgement）和不提前写应答（non Early Write Acknowledgement）。往外设写数据时，cpu 先把数据写入写缓冲区中，若使能了提前应答，则数据到达写缓冲区时就会发送写应答；若没有使能，则数据真正到达外设时才会发送写应答。

linux 内核定义的内存属性

arch/arm64/include/asm/memory.h

内存属性 描述
MT_DEVICE_nGnRnE 设备内存，不聚合，不指令重排，不提前应答
MT_DEVICE_nGnRE 设备内存，不聚合，不指令重排，可提前应答
MT_DEVICE_GRE 设备内存，可聚合，可指令重排，可提前应答
MT_NORMAL_NC 普通内存，关闭高速缓存（Non-Cacheable）
MT_NORMAL 普通内存，打开高速缓存
MT_NORMAL_WT 普通内存，高速缓存的回写策略为 write through

内存属性并没有放到页表项中，而是存放在 MAIR_ELn 寄存器中。页表项中使用一个 3 位（AttrIndx[2:0]）的索引值来查找 MAIR_ELn，然后确定内存属性。

MAIR_ELn 分为 8 个域，每个域表示一种内存属性。
